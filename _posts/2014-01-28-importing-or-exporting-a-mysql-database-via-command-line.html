Many times it is useful to import or export a database via command line. For those of you who have used database management tools such as phpMyAdmin, you are aware how much of a pain it is importing or exporting large databases due to limits in PHP such as max_execution_time.

Importing a MySQL database

mysql -u USERNAME -p DATABASE_NAME < BACKUP_NAME.sql

Note that -p specified that we are going to login with a password. We never specify a password directly due to security reasons, as every command you type is stored in a file called .bash_history.

Exporting a MySQL database

This command is exactly the same as the one used for importing, with two slight differences. We replace < with > and mysql with mysqldump.

mysqldump -u USERNAME -p DATABASE_NAME > BACKUP_NAME.sql
