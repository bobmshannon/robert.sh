Sometimes it may be necessary to run more than one version of jQuery due to compatibility issues. Fortunately, this is a very easy thing to do.

One just needs to include the jQuery libraries like so:

<script src="http://code.jquery.com/jquery-1.8.0.min.js"></script>
<script> var $old = jQuery.noConflict();</script>
<!--INCLUDE & INITIALIZE PLUGINS THAT USE JQUERY 1.8 HERE-->

<script src="http://code.jquery.com/jquery-latest.min.js" type="text/javascript"></script>
<!--INCLUDE & INITIALIZE PLUGINS THAT USE LATEST VERSION OF JQUERY HERE-->

By default, we reference jQuery with the variable name $. By using noConflict(), we can assign jQuery to any variable name that we'd like. In this case we use $old to reference jQuery 1.8, and the default $ to reference the latest version of jQuery.

For example, if we want to initialize a particular plugin using the latest version of jQuery, the code would look something like this:

<script type="text/javascript">
$(document).ready(function() {
    $('.selector').somePlugin();
});	
</script>

If we wanted to initialize this same plugin using the jQuery 1.8, we'd replace $ with $old:

<script type="text/javascript">
$old(document).ready(function() {
    $old('.selector').somePlugin();
});	
</script>

And that's all there is to it. Of course, it is always recommended to try and run only one version of jQuery for simplicity and faster page load times. Not to mention running two versions of jQuery on a web page is well, ugly.
